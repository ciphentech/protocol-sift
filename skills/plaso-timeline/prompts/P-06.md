# Skill: NTP Time Enrichment (ntp_resolver / ntp_enricher)

## Overview
This skill anchors every Plaso event timestamp to NIST UTC by extracting the
NTP sync chain directly from Windows Time Service logs in the artifact. Run
after psort.py exports l2tcsv. Never modifies source evidence.

## Tools
| Tool | Purpose |
|------|---------|
| `python analysis-scripts/ntp_resolver.py` | Resolve NTP source; emit NTPContext |
| `python analysis-scripts/ntp_enricher.py` | Compute fields; write enriched CSV + manifest |

## Workflow

### Step 1 — Locate the psort export
Confirm the l2tcsv export exists before proceeding:
```bash
ls -lh ./exports/<CASE_ID>_timeline.csv
```
If absent, run the plaso-timeline skill first.

### Step 2 — Probe the artifact for NTP logs (ALWAYS run this first)
Scan the desc column before asking the analyst anything:
```bash
grep -c "\[260 " ./exports/<CASE_ID>_timeline.csv || echo "no EID 260"
grep -c "\[259 " ./exports/<CASE_ID>_timeline.csv || echo "no EID 259"
grep -c "\[37 "  ./exports/<CASE_ID>_timeline.csv || echo "no EID 37"
```
If any EID 259, 260, or 37 rows are present: the enricher will extract the
NTP source and offset automatically. Skip Step 3 and proceed to Step 4.
If none are present: proceed to Step 3.

### Step 3 — NTP source resolution (only if Step 2 found no NTP logs)
Ask the analyst ONCE:
  "No Windows Time Service logs found in the artifact.
   Do you know the NTP source for [HOST]?
   Options: (a) provide hostname/IP, (b) unknown — I will assume a default."
Use the answer to populate --ntp-source. If unknown, choose the default:
  - Windows domain-joined → "Domain Controller (assumed)"
  - Windows standalone   → "time.windows.com"
  - AWS Linux/Windows    → "169.254.169.123"
  - Azure Linux/Windows  → "time.windows.com"
  - GCP Linux/Windows    → "metadata.google.internal"
  - Unknown              → "pool.ntp.org"
Set ntp_assumption=true for all defaults. Document the assumption.

### Step 4 — Run enrichment
```bash
python analysis-scripts/ntp_enricher.py \
  --input  ./exports/<CASE_ID>_timeline.csv \
  --output ./exports/<CASE_ID>_timeline_enriched.csv \
  [--ntp-source <value if from Step 3>]
```
The enricher writes ./analysis/<CASE_ID>_ntp_manifest.json on completion.

### Step 5 — Read the manifest and decide
```bash
cat ./analysis/<CASE_ID>_ntp_manifest.json
```
The manifest contains `rubric_pass` (bool) and `rubric_failures` (list).
  - If rubric_pass is true: proceed to Step 7.
  - If rubric_pass is false: read rubric_failures and proceed to Step 6.

### Step 6 — Self-correction (runs only when rubric_pass is false)
Apply the corrective action from `suggested_corrective_action`:

  relax-assumption:
    A default NTP source was used but EID logs were present and inconsistent.
    Re-run Step 2 with grep to confirm which EID provides the source.
    Re-run Step 4 without --ntp-source (let the enricher auto-detect).
    Log: [ntp-enrichment.self-correct] iter=N reason=<failure> action=relax-assumption

  recheck-offset:
    The ntp_offset_s value is outside ±1 s — unusually large for NTP.
    Run: grep "Phase Offset" ./exports/<CASE_ID>_timeline.csv | sort -u
    Confirm the value is genuinely present in the artifact.
    If confirmed: document in ./analysis/<CASE_ID>_ntp_caveats.txt.
    If implausible: re-run with --ntp-source and override offset to 0.

  escalate-to-operator:
    After 3 iterations or contradictory evidence.
    Write ./analysis/<CASE_ID>_ntp_caveats.txt explaining the conflict.
    Do NOT loop further.

Each re-run writes a new manifest as <CASE>_ntp_manifest.iter<N>.json.
Never overwrite a prior manifest.
Cap total iterations at 3. After cap: escalate-to-operator regardless.

Log every iteration to ./analysis/forensic_audit.log:
[ntp-enrichment.self-correct] iter=N reason=<failure> action=<action>

### Step 7 — Verify and summarise
```bash
python -c "
import csv, sys
rows = list(csv.DictReader(open(sys.argv[1])))
print(f'Rows: {len(rows)}')
print(f'NTP source: {rows[0][\"ntp_source\"]}')
print(f'Assumption: {rows[0][\"ntp_assumption\"]}')
print(f'Offset (s): {rows[0][\"ntp_offset_s\"]}')
print(f'nist_time range: {rows[0][\"nist_time\"]} → {rows[-1][\"nist_time\"]}')
" ./exports/<CASE_ID>_timeline_enriched.csv
```
State findings in this format:
  NTP source confirmed: base-dc.shieldbase.lan (EID 260, confidence rank 1)
  Clock offset: -0.0004007 s (≈ 400 µs behind NIST)
  ntp_assumption: false — artifact logs confirmed
  Timeline sorted on nist_time. Ready for cross-source correlation.

## Self-correction
After Step 4, always READ the manifest (Step 5) before stating any findings.
rubric_pass=false is not a failure state — it is a signal to apply Step 6.
Findings stated before reading the manifest are premature and will be wrong.
