## P-09 — Agent trace test: simulate the full reasoning loop

**Goal:** Prove the agent loop works end-to-end. This is the canonical demo
video scene: the agent runs Step 2 (probe), reads the manifest, sees
`rubric_pass=false`, self-corrects to `rubric_pass=true`. Without this test,
you have tools but no agent.

**Prompt:**

````
Work inside protocol-sift/. Create analysis-scripts/tests/smoke_ntp_agent.sh.
This script simulates what the Claude Code agent does: reads manifest,
branches on rubric_pass, re-runs with correction. No actual Claude API calls.
The script IS the agent's decision logic written as shell so it can be
tested deterministically.

Deliverables:

1. tests/fixtures/ntp_mini_no_eids.csv — a 6-row synthetic l2tcsv with NO
   NTP EID rows (only file, registry, and generic WinEVTX rows). Used to
   force the "no logs found" branch and trigger a plausibility failure.

2. analysis-scripts/tests/smoke_ntp_agent.sh
   Simulates the agent's Step 2–6 loop from SKILL.md. Assertions:

   --- SCENE 1: artifact has EID logs (happy path) ---
   (a) Run enricher against ntp_mini.csv (has EID 259/260).
       Read manifest. Assert rubric_pass=true.
       Assert ntp_assumption=false.
       Assert ntp_source="base-dc.shieldbase.lan".
       PASS message: "PASS scene1: EID logs found, rubric_pass=true, no correction needed"

   --- SCENE 2: no EID logs, injected implausible offset (self-correction) ---
   (b) Run enricher against ntp_mini_no_eids.csv with --ntp-source "dc01"
       AND inject an implausible offset by temporarily writing a
       ntp_rubric.json that sets ntp_offset_max_abs_s to 0.00001
       (this will cause the default 0.0 offset to appear out-of-bounds
       for testing — OR use a helper flag --force-offset 500.0 added in
       this step to ntp_enricher.py for testing only).
       Read manifest. Assert rubric_pass=false.
       Assert suggested_corrective_action="recheck-offset".
       PASS message: "PASS scene2-iter1: rubric_pass=false, correction triggered"
   (c) Simulate self-correction: re-run with --ntp-source "dc01" and
       without the injected implausible offset (remove test rubric.json).
       Read iter2 manifest. Assert rubric_pass=true.
       PASS message: "PASS scene2-iter2: after correction, rubric_pass=true"
   (d) Assert two manifest files exist:
       ./analysis/*_ntp_manifest.json  (iter0, rubric_pass=false)
       ./analysis/*_ntp_manifest.iter1.json  (rubric_pass=true)
       PASS message: "PASS scene2-manifests: iter0 and iter1 both exist"

   --- SCENE 3: iteration cap ---
   (e) Simulate 3 iterations all failing (use 3x injected-bad rubric).
       Assert that after iter=3, suggested_corrective_action="escalate-to-operator".
       PASS message: "PASS scene3: iteration cap reached, escalate-to-operator"

   --- SCENE 4: spoliation ---
   (f) Hash fixture before run. Run enricher. Hash after run.
       Assert hashes match.
       PASS message: "PASS scene4: source CSV unchanged (spoliation check)"

   --- SCENE 5: forbidden output path ---
   (g) Run enricher with --output /cases/test.csv.
       Assert exit code non-zero.
       PASS message: "PASS scene5: /cases/ output path rejected"

   Total: 7 PASS lines.

   After all assertions, append to ./analysis/forensic_audit.log:
   [ntp-enrichment.smoke] $(date -u): 7/7 assertions passed

3. Add --force-offset flag to ntp_enricher.py CLI (testing use only):
   p.add_argument('--force-offset', type=float, default=None,
       help='TESTING ONLY: override ntp_offset_s in the returned NTPContext')
   If set, patch ctx.ntp_offset_s = args.force_offset after resolution.
   Emit a warning to stderr: "[ntp-enrichment] WARNING: --force-offset active
   — for testing only, do not use in production"

Acceptance:
````
